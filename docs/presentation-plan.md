# Coritiba Concept Store Presentation Plan

## Goal

Present the project as a premium concept store, using a curated visual deck to create desire first and the live site as the proof of execution.

## Core positioning

Use this framing consistently:

- `Coritiba Concept Store`
- `Premium e-commerce concept`
- `Digital fashion experience`
- `Creative direction, product design, and front-end execution`

Avoid:

- comparing it with any official site
- saying it is "better than"
- framing it as a correction of an existing product

## Recommended publishing stack

### 1. Live site

Primary recommendation: Vercel.

- Why: it is the most direct hosting path for a Next.js project.
- Official docs: [Next.js on Vercel](https://vercel.com/docs/concepts/next.js/overview)

Suggested deploy flow:

1. Publish the code to GitHub.
2. Import the GitHub repository into Vercel.
3. Keep the production domain clean and short.
4. Use the live URL as the final CTA of the presentation.

Suggested final CTA copy:

- `Explore the live concept`
- `View live experience`
- `Open concept store`

### 2. Slide deck

Primary recommendation: Figma Slides.

- Why: best fit for a design-led, editorial, premium deck.
- Official product page: [Figma Slides](https://www.figma.com/slides/)
- Official help article: [Explore Figma Slides](https://help.figma.com/hc/en-us/articles/24170630629911-Explore-Figma-Slides)

Fallback tools:

- Keynote
- Adobe InDesign
- PowerPoint, only if the design is custom-built and not template-looking

### 3. Portfolio / case page

Primary recommendation: Behance.

- Official guide: [Create & Publish a Project](https://help.behance.net/hc/en-us/articles/204483684-Guide-Create-Publish-A-Project)
- Why: great for long-form visual storytelling, project covers, and discovery

### 4. Professional distribution

Primary recommendation: LinkedIn.

- Official thought leadership context: [LinkedIn Thought Leadership](https://business.linkedin.com/marketing-solutions/success/marketing-terms/thought-leadership)

Use LinkedIn for:

- launch announcement
- short project story
- 6-10 still images
- link to live site

## Presentation format

The best format for this project is:

1. premium still-image deck
2. live site linked at the end

This works better than a video-first launch because:

- the deck controls pacing
- each screen gets proper focus
- you frame the concept before people explore it
- the site itself becomes the interactive second act

## Visual rules for the deck

### Layout

- use strong vertical rhythm
- keep generous negative space
- avoid filling every slide with screenshots
- mix full-bleed imagery with detail crops

### Typography

- keep type large and intentional
- use 1 display face + 1 support face at most
- short headlines, not long paragraphs

### Text density

- 1 idea per slide
- 15 to 35 words max on most slides
- let the images do the work

### Image treatment

- use clean device-free crops first
- only add browser frames when you need to prove it is real
- highlight key moments with close crops, not annotations everywhere

### Color

- let the project’s green, black, and white lead the narrative
- keep deck background restrained
- avoid generic gradient presentation styling

- product page
- fit guide modal
- favorites
- cart
- returns page
- tracking page

### Mobile

Capture:

- hero

### Capture rules

- always use the same zoom level
- hide noisy browser chrome when possible
- capture at high resolution
- crop with intention, not just full-page dumps

## Suggested posting order

### Phase 1. Publish infrastructure

1. push repository to GitHub
2. deploy on Vercel
3. test desktop and mobile
4. define final public URL

### Phase 2. Build presentation assets

1. capture screenshots
2. sort them into sections
